# CS 370 Portfolio Reflection

## Pirate Intelligent Agent Project

For this project, I worked on a pirate intelligent agent that used reinforcement learning and neural networks to navigate a maze and find treasure. The project was completed in a Jupyter Notebook and demonstrated how an intelligent agent can learn from experience instead of being directly programmed with every possible move.

Some of the code was provided to me, including the environment setup for the maze, the `TreasureMaze` class, and supporting code that helped manage the game experience and store actions, rewards, and states. This provided the structure for the agent’s world, including the maze layout, valid movements, rewards, and win conditions. The code I created focused on training the intelligent agent. I completed the deep Q-learning training process, including choosing actions, balancing exploration and exploitation, storing experiences, replaying past experiences, updating the neural network, and improving the agent’s ability to reach the treasure over time.

## Connection to Computer Science

Computer scientists solve problems by designing, building, testing, and improving technology-based solutions. Their work matters because software and intelligent systems are used in many areas of life, including business, healthcare, education, transportation, cybersecurity, and entertainment. In this course, I learned that artificial intelligence is not just about making programs complete tasks. It is also about creating systems that can learn from data, adapt to new situations, and make decisions based on patterns.

As a computer scientist, I approach problems by first understanding the goal, the constraints, and the environment. Then I break the problem into smaller parts that can be designed, coded, tested, and improved. In this project, that meant understanding how the maze worked, how the pirate agent received rewards, and how the neural network learned from successful and unsuccessful moves. I also learned the importance of testing and adjusting parameters because small changes in training settings can affect how well an AI model performs.

## Ethical Responsibilities

My ethical responsibilities to the end user include creating systems that are reliable, fair, safe, and understandable. Users should be able to trust that an intelligent system is making decisions in a way that supports their needs and does not cause unnecessary harm. For an organization, my responsibilities include protecting data, reducing risk, and making sure the system performs as intended before it is used in a real-world setting.

This project showed me that AI systems need careful design and testing. Even though the pirate agent was only solving a maze, the same ideas apply to larger real-world AI systems. Developers must think about how decisions are made, whether the model has been trained properly, and what could happen if the system behaves incorrectly. Responsible AI development means balancing performance with safety, transparency, and accountability.
